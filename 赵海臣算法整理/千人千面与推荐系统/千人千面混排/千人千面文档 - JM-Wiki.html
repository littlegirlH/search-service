[编辑] 项目描述

千人千面现在是根据优质用户的行为特征，加上运营选品形成一个最终推荐列表

[编辑] 用户区分

  • 弱账户体系（无uid）
  • 普通用户
  • 优质用户（ 180天内订单 >= 3单 && 90天内浏览 >= 15次）

[编辑] 接口及context参数

Client : recommendService - recommendForUser

接口文档：http://wiki.int.jumei.com/index.php?title=%E9%A6%96%E9%A1%B5%E4%B8%AA%E6%80%A7%E5%8C%96%E6%8E%A8%E8%8D%90%E6%8E%A5%E5%8F%A3%E8%AF%B4%E6%98%8E

[编辑] 调用千人千面流程

  • 当无uid和uid不是优质用户时，调用混排
  • 当uid为优质用户时，调用千人千面接口

是否为优质用户是api通过接口Client : recommendService - getRecommendationSize来判断

分页缓存：

前端操作 API 推荐服务
刷新进入首页 首次调用千人千面接口 生成分页缓存key,缓存全部分页概要数据到Redis，返回当前页数据
往下滑动请求第二页数据 传入分页缓存标识pagination_cache_key 和分页一致性pagination_consistency 根据用户缓存的概要数据去请求商品全量缓存，其中选品和置顶数据是直接调用的商品库数据
强制刷新数据 不传入分页缓存标识pagination_cache_key 重新计算ecpm，合并排序所有数据

推荐内部缓存： 商品全量缓存（2min）----->ecpm计算结果缓存

[编辑] 数据源

  • 商品 
   * 长期模型数据，pair_source:ALS
   * 短期模型数据，pair_source:CF
   * 运营选品，pair_source: DEALMAN_SELECTION_DATA
   * 运营置顶选品，pair_source: DEALMAN_FIXED_DATA
   * dove配置，pair_source: DOVE_CONFIG
   *实时推荐（搜索词，加购，订单），pair_source:rts,rtc,rt
  • 专场 
   * 个性化专场（10个，根据用户画像），pair_source:ACTIVITY_PORTRAITS
   * 运营选择专场，pair_source: DEALMAN_SELECTION_DATA
   * 运营置顶专场，pair_source: DEALMAN_FIXED_DATA

[编辑] 排序

专场和商品都是根据mixed_index_score降序排序

[编辑] 叠加功能

  • 实时推荐 
   监控用户的搜索，加购，下单购买的行为，根据消息进行实时推荐，现在实时推荐方式支持在dove中配置：【recommend.model.strategy.config】
   dove配置context参数如下：
字段名 类型 说明 枚举值
realtime_data_priority string 实时推荐优先级 “shop_cart,processed_order,search_word”
realtime_expiretime_processed_order int 订单行为的实时推荐数据过期时间范围,单位/分 30
realtime_expiretime_search_word int 搜索行为的实时推荐数据过期时间范围,单位/分 30
realtime_expiretime_shop_cart int 加购行为的实时推荐数据过期时间范围,单位/分 30
realtime_fixed_count_processed_order int 订单行为的实时推荐数据量 3
realtime_fixed_count_search_word int 每个搜索词的候选集数量 3
realtime_fixed_count_shop_cart int 加购行为的实时推荐数据量 3
realtime_mix_fixed_index_search_word string 给根据搜索词推荐的数据指定位置 “1,2,3,4”
realtime_mix_type string 实时推荐数据融合方式 "top":按照优先级强制置顶, "fixed": 根据不同数据指定位置,"random":根据realtime_mix_type_extra给出范围的随机指定位置
realtime_mix_type_extra string 当realtime_mix_type==fixed时，作为补充位置，当realtime_mix_type==random时，作为随机范围最大值
realtime_model_enable bool 实时推荐开关
realtime_score_weight_processed_order float 实时推荐订单权重分
realtime_score_weight_shop_cart float 实时推荐加购权重分
realtime_expiretime_search_word_except int 过滤订单来源搜索词的时间,单位分
  • 分类数量限制 
   限制千人千面某一个分类的个数,支持dove配置开关和分类，数量，dove配置如下：【recommend.model.strategy.config】
字段名 类型 说明 枚举值
category_limited_count int 分类数量阈值 3
category_limited_enable bool 分类数量限制开关 打开：true，关闭：false
category_limited_field string 分类限制的作用域 category_id_3，category_id_4等
  • 一起团限制数量和价格 
   根据运营需求，希望限制首页出现低调性的一起团商品，同时限制一起团商品数量，dove配置如下：【recommend.model.strategy.config】
字段名 类型 说明 枚举值
yqt_filter_price float 首页一起团商品最低价，低于这个价不被展示 29.5
yqt_filter_count int 首页展示一起团的数量不超过这个值 3
  • uid指定流量分组 
   为了方便测试，需要指定用户到某一个流量组里，dove配置如下：【recommend.system.common】
   [ab.version.mapping]
   u48504212="b"
  • uid数据映射到指定uid 
   为了方便追踪问题，把问题uid的数据映射到指定uid,dove配置如下：【recommend.system.common】
   [user.mapping]
   #u163216="109325865"
  • 首页选品AB 
   1.选品商品支持预热，不可售
   2.选品来源是老接口Client : clientCmsActMixedSchedule - getActMixedSchedule（默认选品组），新接口Client : clientCmsActTestSchedule - getActTestSchedule（返回分组选品数据，选品的分组对应了流量的分组），dove配置选品数据，当enabled_selection_ab_groups有值时，使用dove配置选品数据，未配置时使用新接口数据
   3.选品曝光到达一定规定数量后（在dealman中配置，通过新接口返回），可以自动下掉商品
  • 支持最后结果过滤选品数据，dove配置如下：【recommend.model.strategy.config】
字段名 类型 说明 枚举值
exclude_selection_data bool 是否过滤选品 过滤：true，不过滤：false
  • 限制个性化数据数量，dove配置如下：【recommend.model.strategy.config】
字段名 类型 说明 枚举值
personaliztion_candidates_limit int 限制个性化数据数量
